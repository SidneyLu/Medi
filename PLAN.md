Medi：家用医疗健康 RAG WebApp 实施计划
Summary
首期交付中国大陆 Web 版：Docker封装 TypeScript Next.js 前端、FastAPI 后端、PostgreSQL、Milvus、Qwen API。
MSD 大众版 https://www.msdmanuals.cn/home 是唯一医学知识来源；内容全量同步、展示原始预览及多媒体。
产品定位为健康科普与安全分诊：仅提供参考治疗方案；检查建议仅作为“与医生讨论”的方向。
Architecture And Knowledge Base
建立授权内容同步服务：抓取/导入 MSD 页面、插图、视频、动画、3D 与测验，保留源 URL、标题、栏目树、发布时间、内容版本、授权范围和内容哈希。
PostgreSQL 保存结构化实体与关系：疾病、症状、检验指标、急救、特殊人群、药物科普、多媒体和测验；对象存储保存授权媒体及原始页面快照。
以章节级语义块写入 Milvus；同时将文本与关键词写入 PostgreSQL 全文索引，采用关键词 + 向量混合召回、重排和栏目过滤。
每个知识块保留原文定位、版本与引用锚点。回答只能使用检索到的块；缺少依据时明确说明并引导用户查看 MSD 原文，不补充外部医学知识。
同步任务采用增量抓取、内容差异检测、版本化索引和人工发布审批；支持单页回滚、过期内容下线和引用可追溯。
Qwen模型 Qwen3.6Plus
Core Product Modules
AI 问答与分诊：文字输入、症状树追问和会话摘要；先由确定性红旗规则识别胸痛、呼吸困难、意识障碍、严重外伤等，再由 Qwen 依据检索结果生成低/中风险的观察、就医科室和就诊准备建议。
输出固定为风险等级、依据、可采取的非诊疗行动、何时立即就医、MSD 原文引用；高风险覆盖普通回答并展示急救步骤和紧急就医提示。
化验单解读：本地 OCR 提取报告字段、单位、数值、参考区间与采样时间；按年龄、性别、生理状态、单位和 MSD 已入库参考值匹配，无法可靠匹配时仅展示原始数据，不判定高低。
报告解释仅引用 MSD 指标科普，明确实验室参考区间差异；允许生成可打印家庭版报告，禁止生成诊断、处方和个体化治疗结论。
图像与多媒体：图片仅用于检索 MSD 相似科普内容及急症警示，不输出疾病识别结果；根据知识条目关联原版图片、动画、3D 模型和视频。
家庭档案：成员、慢病、常用药、血压/血糖/体重/睡眠/用药记录、趋势图、收藏和提醒。提醒规则仅从已版本化的 MSD 条目生成，所有自动提醒均可关闭。
急救中心、分人群专区、知识检索和测验复用统一知识实体、引用与媒体服务；急救内容支持受授权范围约束的离线加密缓存。
Public Interfaces And Safety
FastAPI 提供认证、成员档案、健康记录、知识检索、RAG 对话、报告上传/OCR、媒体预览、收藏、提醒和审计接口；所有 API 使用版本化 /api/v1 契约。
Qwen API 分别承担对话生成、查询改写/重排和视觉辅助；服务端强制结构化输出校验、引用校验、敏感提示注入和失败降级。
分诊红旗规则作为独立、可版本化配置，不依赖 LLM 判断；规则命中、模型输出、检索证据、内容版本和用户确认均写入审计日志。
中国大陆部署：健康档案、报告和图片按敏感个人信息处理，实施单独同意、最小化采集、加密存储与传输、账户注销、数据导出/删除、访问控制、操作审计和保留期限策略。
前端仅提供 Web 响应式体验；原生 App、小程序、硬件同步和端侧模型定义为后续阶段，通过预留设备数据接入与通知接口接续，不纳入首期交付。
Delivery And Validation
阶段一：内容导入、结构化建模、版本化索引、MSD 原文预览和全局检索。
阶段二：带强制引用的 RAG 问答、红旗规则与急救中心，完成医学安全评审。
阶段三：账户/家庭档案、OCR 报告解读、提醒、媒体/测验和离线急救缓存。
测试覆盖：抓取去重与版本回滚、实体链接、混合检索召回、每句医学建议均有有效引用、无证据拒答、红旗症状覆盖、OCR 单位/人群错配、权限隔离、删除导出和并发性能。
上线前由临床专家抽检分诊、急救、检验和特殊人群场景；持续监控无引用回答率、拒答率、红旗漏报/误报、检索命中率、OCR 字段准确率和内容同步失败率。
Assumptions
已取得 MSD 大众版全量正文、原始预览、多媒体、离线缓存及测验内容的书面授权，并将授权地域、署名、缓存、更新和下线条款配置为发布门禁。
Qwen API 可在既定部署与数据合规边界内使用；上传的报告和图片不用于模型训练，除非用户另行明确同意。
医学建议的唯一事实依据为授权入库的 MSD 内容；产品不以“诊断工具”或“替代医生”定位。
